# 思维导图信息获取指南

HeySure AI 的流程编辑器中的 **思维导图信息 (Mindmap Info)** 节点支持通过输入端接收指令，灵活获取思维导图的特定信息。

## 基础功能

该节点可以在属性面板中预设获取的信息类型：
- **完整导图(Markdown)**: 获取整张导图的 Markdown 结构。
- **仅导图结构**: 同上。
- **导图摘要(一级节点)**: 获取中心主题和一级分支名称。
- **仅中心主题**: 仅获取根节点名称。
- **完整导图(JSON)**: 获取导图的完整 JSON 数据。
- **导图文件列表**: 获取当前所有思维导图文件的列表。

## 动态指令输入

除了预设类型，你还可以通过输入端口发送特定文本指令，动态获取信息。

### 1. 获取导图文件列表
发送以下任一指令可获取所有思维导图的列表（包含名称和 ID）：
- `files`
- `list`
- `ls`
- `导图列表`
- `文件列表`

**示例输入**: `list`
**示例输出**:
```
思维导图文件列表:
- 需求分析 (ID: map_123...)
- 技术架构 (ID: map_456...)
```

### 2. 获取指定节点及其子节点
发送节点名称即可获取该节点及其直接子节点的信息（默认深度为 1）。

**语法**: `node:<节点名称>` 或 `节点:<节点名称>`
**简写**: 如果输入内容完全匹配当前导图中的某个节点名称，也会自动触发查询。

**示例输入**: `node:后端开发`
**示例输出**:
```
节点 "后端开发" 的子节点 (1级):
后端开发
├─ API设计
├─ 数据库建模
└─ 鉴权服务
```

### 3. 指定查询深度
结合节点查询，可以指定获取子节点的层级深度。

**语法**: `depth:<层级数>` 或 `深度:<层级数>` 或 `level:<层级数>`

**示例输入**: `node:后端开发 depth:2`
**示例输出**:
```
节点 "后端开发" 的子节点 (2级):
后端开发
├─ API设计
│  ├─ RESTful规范
│  └─ GraphQL
├─ 数据库建模
...
```
 
### 4. 跨导图查询
即使当前未打开目标导图，也可以通过指令指定要查询的导图。

**语法**: `map:<导图名称>` 或 `导图:<导图名称>`

**示例输入**: `map:需求分析 node:用户模块`
**说明**: 这将从 "需求分析" 导图中查找 "用户模块" 节点并返回信息。

## 组合指令示例

你可以组合使用上述指令来精确获取信息。

**场景**: 获取 "产品规划" 导图中 "Q1目标" 节点下的所有层级内容。
**输入**: `map:产品规划 node:Q1目标 depth:99`

**场景**: 仅查看当前导图 "核心功能" 节点的直接子级。
**输入**: `核心功能` (假设 "核心功能" 是存在的节点名)
